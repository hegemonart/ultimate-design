# Plan 11-04: Discussant Question-Quality Feedback Loop + Global Skills Layer

**Phase**: 11 — Self-Improvement  
**Requirements**: SLF-07 (discussant question-quality), SLF-08 (global skills layer)  
**Depends on**: Plan 11-01 (design-reflector exists), Plan 11-03 (/gdd:apply-reflections exists)

## Goal

Two deliverables: (1) teach the discussant to log answer quality, and teach the reflector to detect low-value questions. (2) ship the global skills layer at `~/.claude/gdd/global-skills/` with a promotion flow from project-local to global.

## Tasks

### Task 04-A — Extend `agents/design-discussant.md` with answer-quality logging

Read `agents/design-discussant.md`. Add a new section: **Answer Quality Logging**.

After each question-answer exchange, the discussant appends a quality record to `.design/learnings/question-quality.jsonl` (creating the file if it doesn't exist):

```json
{"ts":"<iso>","question_id":"Q-01","question_text":"<text>","answer_summary":"<one-line>","quality":"high|medium|low|skipped","evidence":"<why low — e.g. user said 'I don't know', answer overridden in D-15, answer length < 10 words>","cycle":"<slug>"}
```

Quality classification rules (apply automatically, no user interaction):
- **skipped**: user typed "skip", "n/a", "pass", "doesn't matter", or pressed Enter with empty input
- **low**: answer was < 10 words AND not a specific value (not a hex code, number, or named token); OR the answer was immediately contradicted by a later D-XX decision in the same session
- **medium**: answer was ≥ 10 words but contained hedging ("maybe", "probably", "I think", "not sure")
- **high**: specific, actionable answer with no hedging

This logging is append-only. The reflector reads it in the **Discussant Question Quality** section.

### Task 04-B — Extend `agents/design-reflector.md` with question-quality analysis

Add section **Discussant Question Quality** to the reflector agent body.

Logic:
1. Read `.design/learnings/question-quality.jsonl` (if exists)
2. Aggregate per `question_id`: count `skipped`, `low`, `medium`, `high` across all cycles
3. Flag questions where (`skipped` + `low`) / total_asks > 0.6 across ≥3 cycles
4. For each flagged question, emit a `[QUESTION]` proposal:

```
### Proposal N — [QUESTION] Prune "What is your preferred animation easing?" 
**Why**: Q-07 got quality=low or skipped in 5 of 6 asks (cycles 1–4)
**Change**: Remove question Q-07 from agents/design-discussant.md question pool.
  Alternative: reword as "Do you use CSS easing presets? (yes/no)" for faster answer.
**Risk**: low
```

If `question-quality.jsonl` doesn't exist: skip and note "question quality data not found — requires at least one discuss session with Phase 11 discussant."

### Task 04-C — Create global skills directory scaffold

Create the global skills directory and README:

**File**: `~/.claude/gdd/global-skills/README.md`

Content:
```markdown
# GDD Global Skills

Cross-project design decisions and conventions that apply to all projects.

Files in this directory are auto-loaded by get-design-done's explore, plan, and design stages (alongside project-local `./.claude/skills/design-*-conventions.md`).

## How to populate
Use `/gdd:apply-reflections` and select a `[GLOBAL-SKILL]` proposal to promote a project finding here.

## File naming
`design-<topic>-conventions.md` — conventions and preferences (always use X)
`design-<topic>-findings.md` — empirical findings (X works better than Y for Z reason)

## Example files
- `design-color-conventions.md` — "I always use oklch(), never sRGB hex in design tokens"
- `design-typography-conventions.md` — "I always start with Inter as the base, customize from there"
- `design-motion-conventions.md` — "Always respect prefers-reduced-motion; use framer-motion API for spring physics"
```

Note: this directory lives in the user's home Claude config (`~/.claude/gdd/global-skills/`), not in the plugin repo. The plugin references it by convention; the directory itself is owned by the user.
