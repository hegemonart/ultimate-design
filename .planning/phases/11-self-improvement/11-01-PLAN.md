# Plan 11-01: design-reflector Agent + /gdd:reflect Command

**Phase**: 11 — Self-Improvement  
**Requirements**: SLF-01 (design-reflector agent), SLF-02 (/gdd:reflect command + /gdd:audit integration)  
**Depends on**: Phase 10 (`.design/intel/`, `.design/learnings/`), Phase 10.1 (`.design/telemetry/costs.jsonl`, `.design/agent-metrics.json`)

## Goal

Ship the `design-reflector` agent — the core of the self-improvement loop. The agent reads cycle artifacts and produces `.design/reflections/<cycle-slug>.md` with actionable proposals. Wire it into `/gdd:audit` (runs automatically at cycle end) and expose `/gdd:reflect` for on-demand runs.

## Tasks

### Task 01-A — Create `agents/design-reflector.md`

Write the agent markdown at `agents/design-reflector.md`.

Frontmatter:
```yaml
name: design-reflector
description: Post-cycle reflection agent. Reads .design/intel/, .design/learnings/, telemetry, and agent-metrics to produce .design/reflections/<cycle-slug>.md with concrete improvement proposals. Spawned by /gdd:audit (end-of-cycle) and /gdd:reflect (on-demand).
tools: Read, Write, Bash, Grep, Glob
color: purple
model: inherit
size_budget: XL
parallel-safe: never
typical-duration-seconds: 60
reads-only: false
writes:
  - ".design/reflections/*.md"
```

Agent body (in the markdown body after frontmatter):

**Role**: You are a post-cycle reflection agent. You analyze what happened in a design cycle, compare outcomes to costs, and produce concrete, reviewable proposals — not generic advice. Every output you write is a proposal the user will review and selectively apply. You never auto-apply anything.

**Required Reading** (supplied by orchestrating stage):
- `.design/STATE.md` — cycle identity, decisions, session history
- `.design/DESIGN-VERIFICATION.md` — cycle outcome scores + gaps
- `.design/learnings/*.md` — structured learnings from Phase 10 extract
- `.design/telemetry/costs.jsonl` — per-agent-spawn cost data
- `.design/agent-metrics.json` — aggregated agent performance data
- `.design/cycles/<slug>/CYCLE-SUMMARY.md` (if present)

If any file doesn't exist: skip that section, note the missing source, and continue.

**Output**: `.design/reflections/<cycle-slug>.md`

**Reflection Sections**:

1. **What surprised us** — decisions that deviated from the plan, unexpected cost spikes, agents that took 3× typical-duration
2. **Recurring decisions** — D-XX codes that appeared in multiple cycles or multiple sessions
3. **Agent performance** — for each agent in agent-metrics.json: compare `typical-duration-seconds` vs measured, flag tier mismatches (Haiku-capable work done at Sonnet tier), flag parallel-safe errors
4. **Anti-pattern recurrence** — patterns flagged in learnings that appeared ≥2 cycles
5. **Low-value discussant answers** — questions where the answer was "don't know", "doesn't matter", or was immediately overridden in the next session
6. **Proposals** — concrete, numbered. Each proposal is tagged with its type:
   - `[FRONTMATTER]` — change to an agent's frontmatter field
   - `[REFERENCE]` — addition to a reference/ file
   - `[BUDGET]` — change to .design/budget.json
   - `[QUESTION]` — prune or reword a discussant question
   - `[GLOBAL-SKILL]` — promote a project-local finding to global

**Proposal format** (each must follow this shape):
```
### Proposal N — [TYPE] Short title
**Why**: (evidence — cite cycle slug, cost figure, or D-XX code)
**Change**: (exact diff — what field/line changes to what value)
**Risk**: low / medium (low = cosmetic; medium = changes agent behavior)
```

**Dry-run mode**: if `--dry-run` flag is passed in the spawning prompt, write proposals to stdout only — do not write `.design/reflections/` file.

**Discipline**: every proposal must cite specific evidence (cost figure, learnings file, D-XX code, cycle slug). Vague proposals ("the agent seems slow") are not valid output.

### Task 01-C — Create `skills/reflect/SKILL.md`

Create the `/gdd:reflect` skill at `skills/reflect/SKILL.md`.

Frontmatter:
```yaml
name: gdd-reflect
description: "Run design-reflector on demand — produces .design/reflections/<cycle-slug>.md with improvement proposals. Review proposals with /gdd:apply-reflections."
argument-hint: "[--dry-run] [--cycle <slug>]"
tools: Read, Write, Task
```

Skill body:
- Parse `--dry-run` and `--cycle <slug>` args
- Determine cycle slug: if `--cycle` given use it; else read current cycle from `.design/STATE.md`
- Verify `.design/STATE.md` exists; abort if not
- Spawn `design-reflector` agent via Task tool, passing: cycle slug, `--dry-run` flag, required reading file list
- After completion, read `.design/reflections/<cycle-slug>.md` (unless dry-run)
- Print summary: number of proposals, proposal types (FRONTMATTER/REFERENCE/BUDGET/QUESTION/GLOBAL-SKILL counts)
- Print next action: "Review proposals with `/gdd:apply-reflections`"
- Dry-run: print proposals to stdout, skip file write
