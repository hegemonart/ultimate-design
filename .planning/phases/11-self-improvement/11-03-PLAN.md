# Plan 11-03: Reference-Update Proposer + /gdd:apply-reflections Command

**Phase**: 11 — Self-Improvement  
**Requirements**: SLF-05 (reference-update proposer), SLF-06 (/gdd:apply-reflections)  
**Depends on**: Plan 11-01 (design-reflector exists), Phase 10 (`.design/learnings/`)

## Goal

Add N-cycle pattern detection to `design-reflector` that drafts additions to `reference/` files when the same gap appears ≥3 times. Ship `/gdd:apply-reflections` — the user-review command that diffs proposals and lets the user selectively apply them.

## Tasks

### Task 03-A — Extend `agents/design-reflector.md` with reference-update proposer logic

Add a new section to the agent body: **Reference Update Proposals**.

Logic:
1. Read all `.design/learnings/*.md` files (from Phase 10's `/gdd:extract-learnings`)
2. Parse learnings for recurring patterns: anti-patterns, heuristics, missing reference knowledge
3. Count pattern recurrence across cycle files. A "pattern" is recurring if the same keyword cluster (e.g., "oklch", "contrast ratio", "motion preference", "skeleton loader") appears in ≥3 distinct learnings files
4. For each pattern meeting the N≥3 threshold:
   - Classify: is this an anti-pattern (don't do X), a heuristic (prefer X when Y), or general reference (X is how we do Z here)?
   - Draft the addition text
   - Identify the target file: `reference/anti-patterns.md`, `reference/heuristics.md`, or propose a new `reference/<topic>.md`
5. Emit a `[REFERENCE]` proposal for each

**Proposal format**:
```
### Proposal N — [REFERENCE] Add oklch color space guidance to heuristics.md
**Why**: "oklch" appeared in learnings files for cycle-1, cycle-2, cycle-4 — always flagged as missing from design decisions
**Change**: Append to reference/heuristics.md:
  > **oklch colors**: Always specify colors in oklch() for P3-wide-gamut support. Fallback to sRGB hex only in legacy context blocks.
**Risk**: low
```

N threshold is configurable: default 3, overridable by a `REFLECTOR_PATTERN_THRESHOLD` env var or `.design/config.json` key `reflector.pattern_threshold`.

If fewer than 3 learnings files exist, skip reference proposals and note: "insufficient cycle history for pattern detection (need ≥3 learnings files, found N)."

### Task 03-B — Create `skills/apply-reflections/SKILL.md`

Create `/gdd:apply-reflections` skill at `skills/apply-reflections/SKILL.md`.

Frontmatter:
```yaml
name: gdd-apply-reflections
description: "Review and selectively apply proposals from .design/reflections/<cycle-slug>.md. Diffs each proposal, prompts user to accept/skip/edit, then writes changes."
argument-hint: "[--cycle <slug>] [--filter <FRONTMATTER|REFERENCE|BUDGET|QUESTION|GLOBAL-SKILL>] [--dry-run]"
tools: Read, Write, Edit, Bash, Glob
```

Skill body:

**Resolve reflections file**:
- If `--cycle` given: load `.design/reflections/<slug>.md`
- Else: list `.design/reflections/*.md` sorted by modified time, load the most recent
- If no reflections file found: error "No reflections found. Run `/gdd:reflect` first."

**Filter proposals**: if `--filter` given, show only proposals matching that type tag. Otherwise show all.

**For each proposal**:
1. Print the full proposal block (Why / Change / Risk)
2. If `--dry-run`: print `[dry-run] would apply` and continue
3. Else: prompt user with options:
   - `a` — apply this proposal
   - `s` — skip (mark as reviewed/skipped in the reflections file)
   - `e` — edit before applying (open the Change text in an inline editor)
   - `q` — quit (stop processing; remaining proposals stay unreviewed)

**Applying a `[FRONTMATTER]` proposal**:
- Find the target agent file (e.g., `agents/design-verifier.md`)
- Read current frontmatter value
- Edit the specific frontmatter line using Edit tool
- Append to reflection file: `**Applied**: 2026-04-18 by user`

**Applying a `[REFERENCE]` proposal**:
- Find or create the target reference file
- Append the drafted text
- Append to reflection file: `**Applied**: date`

**Applying a `[BUDGET]` proposal**:
- Read `.design/budget.json`
- Update the specified key
- Write updated JSON
- Append to reflection file: `**Applied**: date`

**Applying a `[QUESTION]` proposal**:
- Find the target question in `agents/design-discussant.md`
- Edit or remove as specified
- Append to reflection file: `**Applied**: date`

**Applying a `[GLOBAL-SKILL]` proposal**:
- Copy the skill content to `~/.claude/gdd/global-skills/<skill-name>.md`
- Create the directory if it doesn't exist
- Append to reflection file: `**Applied**: date`

**After all proposals processed**: print summary — N applied, N skipped, N remaining unreviewed.
