## Checker Blocker-Fix Verification

The original draft of this plan had a bug: it embedded the dismissal block as `python3 -c '...' CONFIG_PATH=... LATEST_TAG=...` — trailing `KEY=VALUE` after `python3 -c`, which Python treats as `sys.argv[1:]`, NOT environment variables. Accessing them via `os.environ['CONFIG_PATH']` in that form raises `KeyError`, so the config write would never happen and the dismissal would silently fail.

The shipped form is env-prefix: `CONFIG_PATH="$CONFIG_PATH" LATEST_TAG="$LATEST_TAG" python3 <<'PY' ... PY`. The shell places the values into the child process environment before `python3` starts, and the heredoc body reads them via `os.environ['KEY']`. This is the only portable way to pass shell-provided string inputs into a heredoc-embedded python block without shell substitution inside the python body (which would open a quoting/injection surface).

Grep verifications:

```text

    CONFIG_PATH="$CONFIG_PATH" LATEST_TAG="$LATEST_TAG" python3 <<'PY'

$ grep 'python3 -c.*CONFIG_PATH=' skills/check-update/SKILL.md

$ grep -n '^import json, os' skills/check-update/SKILL.md
```
