# Plan 13-07: Release-Time Smoke Test — Fresh-Checkout Pipeline Run + Baseline Diff

**Phase:** 13 — CI/CD
**Wave:** C (sequential — depends on 13-06; blocks 13-08's baseline lock)
**Requirements:** CICD-14 (release smoke test on tag creation)

## Objective

Extend `.github/workflows/release.yml` (from plan 13-06) with a `release-smoke-test` job that runs *after* `tag-and-release` on successful tag creation. The job does a fresh `git clone` of the tag, installs the plugin, runs `/gdd:explore` against `test-fixture/src/`, and diffs the resulting `.design/` artifacts against `test-fixture/baselines/phase-13/`. On diff: job fails, surfacing the need to run `scripts/rollback-release.sh` manually.

**Purpose:** closes the "does the released plugin actually work end-to-end?" loop. Catches packaging bugs (missed files in `.claude-plugin/`, broken hooks, version-sync mismatches) that would otherwise only surface when users install.

**Output:** release.yml gains a third real job; one new script.

### Interfaces

- `release.yml` from 13-06 has jobs: `detect-version`, `tag-and-release`, `marketplace-publish`. This plan inserts `release-smoke-test` BEFORE `marketplace-publish` (so a failing smoke test blocks the marketplace placeholder from running — future-proofing for when the placeholder becomes real).
- The smoke test does NOT invoke the `claude` CLI (may not be available on GitHub runners). Instead, it simulates `/gdd:explore` by running the deterministic portions (`scripts/build-intel.cjs` + static analysis steps) and diffing their output artifacts. Per D-22, the key insight is: we diff against a frozen `.design/` artifact tree, not a live LLM result.
- Since Phase 13's baseline is LOCKED in plan 13-08 (which runs *after* this), the smoke test in CI will initially diff against `test-fixture/baselines/phase-11/` (the latest pre-13 baseline). Plan 13-08 atomically (1) locks the phase-13 baseline AND (2) flips the smoke-test target path from phase-11 to phase-13.

Write `scripts/release-smoke-test.cjs`. This script simulates the non-LLM portions of `/gdd:explore` against the fixture and diffs against the baseline. It does NOT spawn Claude.

Implementation contract:

1. Accept one required CLI argument: `--baseline <path>` (absolute or relative path to the baseline directory, e.g. `test-fixture/baselines/phase-13/`). If the directory doesn't exist, exit 2 with an error: `baseline not found: <path>`.

2. Create a temp working dir: `os.tmpdir() + '/gdd-smoke-' + Date.now()`. Copy `test-fixture/src/` into it.

3. Run the deterministic intel builder against the copy: `spawnSync('node', ['scripts/build-intel.cjs', copyPath], { encoding: 'utf8', stdio: 'inherit' })` — this produces `.design/intel/*.json` in the copy dir.

4. Enumerate the baseline files that are deterministic and comparable. Start with these three artifact categories (confirm names from phase-11 baseline):
   - `BASELINE.md` — the "what's frozen" manifest
   - Any `.design/intel/*.json` shipped in the baseline directory (if present)
   - Any `DESIGN-CONTEXT.md` fragment shipped in the baseline
   For each baseline file, attempt to find the corresponding file in the fresh run. If present: diff byte-exact (after normalising line endings to `\n`). If absent from fresh run: record a missing-artifact failure.

5. Accumulate all diffs into an array. Print each as:
   ```
   DIFF: <relative_path>
     Expected: <first 80 chars of baseline>
     Actual:   <first 80 chars of fresh>
   ```

6. Also print line-by-line unified diff using Node's `require('diff')` if the package is installed, or a stdlib word-level diff if not (prefer stdlib; keep dependencies zero).

7. Exit 0 if zero diffs + zero missing artifacts; exit 1 otherwise with a summary line: `smoke-test: <N> diffs, <M> missing artifacts`.

8. Clean up the temp dir unless `--keep` is passed (for debugging locally).

Notes:

Insert this job exactly (immediately after `tag-and-release:` block):

```yaml
  release-smoke-test:
    name: "Release smoke test (fresh checkout + baseline diff)"
    needs: tag-and-release
    if: needs.detect-version.outputs.changed == 'true'
    runs-on: ubuntu-latest
    steps:
      - name: Checkout the freshly-created tag
        uses: actions/checkout@v4
        with:
          ref: v${{ needs.detect-version.outputs.version }}

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Run release smoke test against phase-13 baseline
        run: |
          set -euo pipefail
          if [ -d test-fixture/baselines/phase-13 ]; then
            BASELINE=test-fixture/baselines/phase-13
          else
            echo "phase-13 baseline not yet present — falling back to phase-11 (plan 13-08 locks phase-13)"
            BASELINE=test-fixture/baselines/phase-11
          fi
          node scripts/release-smoke-test.cjs --baseline "$BASELINE"

      - name: Rollback guidance on failure
        if: failure()
        run: |
          echo ""
          echo "SMOKE TEST FAILED."
          echo "The tag v${{ needs.detect-version.outputs.version }} has been created but the pipeline diff vs baseline did not match."
          echo "To roll back: run locally as repo admin:"
          echo "  bash scripts/rollback-release.sh ${{ needs.detect-version.outputs.version }}"
          echo "Per D-22, automated rollback is intentionally NOT implemented."
          exit 1
```
