2. Write `scripts/run-injection-scanner-ci.cjs`. This must:
   - Enumerate target files using `fs.readdirSync` + `path.join`:
     - All files under `reference/` matching `*.md` (recursively)
     - All `skills/**/SKILL.md` (recursive)
     - All `agents/*.md`
   - For each file, call into the injection scanner. The cleanest approach: `require('../hooks/gdd-read-injection-scanner.js')` and call its exported function (inspect the file during Read step to confirm the export surface). If the scanner is only invocable as a hook (reads stdin), spawn it as `child_process.spawnSync('node', ['hooks/gdd-read-injection-scanner.js'], { input: JSON.stringify({ tool_name: 'Read', tool_input: { file_path: absPath } }), encoding: 'utf8' })`.
   - Accumulate findings. Print one line per finding: `{file}: {finding_type}: {excerpt}`.
   - At end, print `summary: <N> files scanned, <M> findings`.
   - Exit 0 on zero findings, 1 otherwise.
   - Accept no CLI args (paths are hardcoded per D-13). If a `--paths` arg is given in the future, it would override — out of scope for this plan.

Locate `security:` and `size-budget:` jobs. Replace their `steps:` with:

```yaml
  security:
    name: "Security (secrets + injection scan)"
    needs: [test]
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0  # full history for gitleaks initial scan per D-12

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Gitleaks scan
        uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITLEAKS_ENABLE_COMMENTS: false
          GITLEAKS_CONFIG: .gitleaks.toml

      - name: Injection scanner (CI mode)
        run: npm run scan:injection

  size-budget:
    name: "Size budget (blocking)"
    needs: [test]
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Agent size-budget gate
        run: |
          if ! npm run test:size-budget; then
            echo ""
            echo "ERROR: one or more agents exceed their size_budget tier."
            echo "To raise: set size_budget: XL in the agent frontmatter AND include rationale in the PR description."
            echo "See CONTRIBUTING.md for the override process."
            exit 1
          fi
```

Per D-12: `fetch-depth: 0` ensures gitleaks sees full history on the initial run (first-time scan). Subsequent runs on the same branch incremental-scan automatically per gitleaks-action defaults.
Per D-13: injection scanner is a plain npm invocation — the `scripts/run-injection-scanner-ci.cjs` script handles all enumeration.
Per D-14: size-budget gate runs in its own job (not under `test:`) so failures are visually distinct and the error message points to the override process.
