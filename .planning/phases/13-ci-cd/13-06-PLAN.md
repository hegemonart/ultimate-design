# Plan 13-06: Release Automation — Auto-Tag + GitHub Release

**Phase:** 13 — CI/CD
**Wave:** C (sequential — must follow 13-05; blocks 13-07)
**Requirements:** CICD-13 (auto-tag + GitHub Release on plugin.json version change)

## Objective

Ship `.github/workflows/release.yml` — a push-to-main-triggered workflow that detects a `.claude-plugin/plugin.json` version-field change, extracts the matching `## [<version>]` section from `CHANGELOG.md`, creates a git tag `v<version>`, and creates a GitHub Release via `softprops/action-gh-release@v2`. Ship `scripts/extract-changelog-section.cjs` (the parser) and `scripts/rollback-release.sh` (the manual rollback).

**Purpose:** automate the otherwise-manual tag + release step; eliminate drift between plugin.json version and git tags.

**Output:** three new files; one package.json scripts entry added.

1. Write `scripts/extract-changelog-section.cjs` — Node CJS CLI that:
   - Reads `CHANGELOG.md` from repo root.
   - Accepts one CLI argument: the version string (e.g. `1.0.7`).
   - Finds the line matching `^## \\[<version>\\]` (escape brackets). Start capturing from the NEXT line.
   - Stops capturing at the next line matching `^## \\[` (the next version's heading) OR at a line that is exactly `---` (horizontal rule between entries).
   - Strips leading + trailing blank lines from the captured block.
   - Prints the block to stdout. Exits 0 if block was found; exits 1 with `ERROR: no section for version <version>` if not.
   - Rejects empty version argument with exit 2 + usage message.
   - About 40 lines total. Pure stdlib.

2. Write `scripts/rollback-release.sh`:

   ```bash
   #!/usr/bin/env bash
   # rollback-release.sh — manually delete a tag + GitHub Release. NOT called from CI.
   # Usage: bash scripts/rollback-release.sh <version>
   # Example: bash scripts/rollback-release.sh 1.0.7

   set -euo pipefail

   VERSION="${1:-}"
   if [ -z "$VERSION" ]; then
     echo "Usage: $0 <version>"
     echo "Example: $0 1.0.7"
     exit 1
   fi

   TAG="v${VERSION}"
   REPO="${GITHUB_REPOSITORY:-hegemonart/get-design-done}"

   if ! command -v gh >/dev/null 2>&1; then
     echo "ERROR: gh CLI not found."
     exit 1
   fi

   echo "About to delete release + tag $TAG on $REPO."
   echo "This is a DESTRUCTIVE operation. Confirm (y/N): "
   read -r CONFIRM
   if [ "$CONFIRM" != "y" ]; then
     echo "Aborted."
     exit 0
   fi

   # Delete the GitHub Release (this also deletes the tag ref on github.com).
   gh release delete "$TAG" --repo "$REPO" --yes --cleanup-tag || {
     echo "Release delete failed or no release existed; attempting tag-only delete."
     gh api -X DELETE "repos/${REPO}/git/refs/tags/${TAG}" || true
   }

   # Also delete locally if the user has the tag.
   git tag -d "$TAG" 2>/dev/null || true

   echo "Rollback complete for $TAG."
   echo "Note: clones that already pulled the tag retain it. Consider posting"
   echo "      a deprecation note in the next CHANGELOG entry."
   ```

   Make executable: `chmod +x scripts/rollback-release.sh`.

Create `.github/workflows/release.yml` with this exact content:

```yaml
name: Release

on:
  push:
    branches: [main]
    paths:
      - '.claude-plugin/plugin.json'
      - 'CHANGELOG.md'

# Per D-19: workflow only tags + releases. It does not push to main, force-push, or delete refs.
# Per D-22: release-time smoke test is a separate workflow/job — this workflow ends at release creation.

permissions:
  contents: write   # needed to create tags + releases

jobs:
  detect-version:
    name: "Detect version bump"
    runs-on: ubuntu-latest
    outputs:
      version: ${{ steps.diff.outputs.version }}
      changed: ${{ steps.diff.outputs.changed }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 2   # needs HEAD^ for diff

      - name: Detect plugin.json version change
        id: diff
        run: |
          set -euo pipefail
          OLD=$(git show HEAD^:.claude-plugin/plugin.json 2>/dev/null | node -e 'let d=""; process.stdin.on("data",c=>d+=c); process.stdin.on("end",()=>{try{console.log(JSON.parse(d).version)}catch{console.log("")}})' || echo "")
          NEW=$(node -e 'console.log(JSON.parse(require("fs").readFileSync(".claude-plugin/plugin.json","utf8")).version)')
          echo "old=$OLD"
          echo "new=$NEW"
          if [ "$OLD" != "$NEW" ] && [ -n "$NEW" ]; then
            echo "version=$NEW" >> "$GITHUB_OUTPUT"
            echo "changed=true" >> "$GITHUB_OUTPUT"
            echo "Version bump detected: $OLD -> $NEW"
          else
            echo "changed=false" >> "$GITHUB_OUTPUT"
            echo "No version change. Exiting quietly."
          fi

  tag-and-release:
    name: "Tag + GitHub Release"
    needs: detect-version
    if: needs.detect-version.outputs.changed == 'true'
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Extract CHANGELOG section
        id: changelog
        run: |
          set -euo pipefail
          VERSION="${{ needs.detect-version.outputs.version }}"
          BODY=$(node scripts/extract-changelog-section.cjs "$VERSION")
          {
            echo "body<<GDDEOF"
            echo "$BODY"
            echo "GDDEOF"
          } >> "$GITHUB_OUTPUT"

      - name: Create tag
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          VERSION="${{ needs.detect-version.outputs.version }}"
          TAG="v${VERSION}"
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          # Create and push the tag pointing at the merge commit on main.
          git tag -a "$TAG" -m "Release $TAG"
          git push origin "$TAG"

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: v${{ needs.detect-version.outputs.version }}
          name: v${{ needs.detect-version.outputs.version }}
          body: ${{ steps.changelog.outputs.body }}
          draft: false
          prerelease: false
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  marketplace-publish:
    name: "Marketplace publish (placeholder)"
    needs: tag-and-release
    if: needs.detect-version.outputs.changed == 'true'
    runs-on: ubuntu-latest
    steps:
      - name: Placeholder — D-23
        run: echo "no-op — marketplace registry pending; webhook wired here when available"
```

Per D-19: trigger is `push` to `main` with path filter for plugin.json + CHANGELOG.md; detect-version job diffs against HEAD^ to confirm the version field actually changed (mere CHANGELOG edits don't trigger a release).
Per D-20: `softprops/action-gh-release@v2` for the Release creation.
Per D-21: body comes from `scripts/extract-changelog-section.cjs` via a multi-line `GITHUB_OUTPUT` heredoc (`<<GDDEOF` — custom delimiter to avoid collision with CHANGELOG content).
Per D-23: `marketplace-publish` is a placeholder no-op that logs a note.
