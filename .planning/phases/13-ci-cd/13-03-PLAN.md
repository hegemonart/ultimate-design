Locate the `validate:` job. Replace its `steps:` placeholder with this exact sequence:

```yaml
  validate:
    name: "Validate (schemas + plugin + shellcheck)"
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js 22
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Validate JSON schemas
        run: node scripts/validate-schemas.cjs

      - name: Plugin validate (schema-only fallback)
        run: |
          # Attempt claude-code plugin validate if the CLI exposes it; else the schema
          # validation above is our authoritative check per D-09.
          if command -v claude >/dev/null 2>&1 && claude plugin --help 2>&1 | grep -q 'validate'; then
            echo "claude CLI detected — running claude plugin validate ."
            claude plugin validate . || exit 1
          else
            echo "claude plugin validate CLI not available on runner — schema validation (previous step) is the authoritative check per D-09."
          fi

      - name: Shellcheck
        uses: ludeeus/action-shellcheck@master
        with:
          severity: error
          scandir: scripts

      - name: Detect hardcoded absolute paths
        run: |
          set -e
          # D-11: flag /Users/, /home/, C:\ in shipped content directories.
          # Exit nonzero on any match. Exclude DEPRECATIONS.md example strings and this workflow itself.
          MATCHES=$(grep -rnE '/Users/|/home/[a-z]+/|C:\\\\' \
            scripts/ reference/ agents/ skills/ \
            --include='*.sh' --include='*.cjs' --include='*.js' --include='*.md' \
            --exclude-dir='baselines' \
            2>/dev/null | grep -v 'DEPRECATIONS.md' || true)
          if [ -n "$MATCHES" ]; then
            echo "Hardcoded absolute paths detected:"
            echo "$MATCHES"
            echo "Use \$HOME or a relative path instead (Phase 1 cross-platform discipline)."
            exit 1
          fi
          echo "No hardcoded absolute paths found."
```

Per D-06: `validate:schemas` is the authoritative JSON check.
Per D-09: `claude plugin validate` runs conditionally — if CLI is present, it runs; else the schema step above is the fallback. The CI step never fails just because the CLI is absent.
Per D-10: `ludeeus/action-shellcheck@master` at severity=error, scoped to `scripts/`.
Per D-11: grep for `/Users/`, `/home/<user>/`, and `C:\` across shipped dirs; exit nonzero on match.
