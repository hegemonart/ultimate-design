# Plan 13-02: Markdown + JSON Lint, Frontmatter Validator, Stale-Ref Detector

**Phase:** 13 — CI/CD
**Wave:** A (parallel with 13-01, 13-03)
**Requirements:** CICD-03 (markdown + link lint), CICD-04 (JSON schema validation), CICD-05 (frontmatter + stale-ref validation)
**Depends on:** nothing (authors new files)

## Objective

Author all lint/validation tooling as standalone npm scripts that CI jobs invoke. Land five JSON schemas (`reference/schemas/*.json`), two custom Node scripts (`validate-frontmatter.cjs`, `detect-stale-refs.cjs`), one ajv wrapper (`validate-schemas.cjs`), and root configs (`.markdownlint.jsonc`, `.lycheeignore`). Register all as `package.json` scripts so the ci.yml lint + validate jobs added by plan 13-01 can call them.

**Purpose:** the tools must exist before ci.yml can invoke them. A Wave-A sibling (13-01) creates the job skeleton in parallel; when both land, a follow-up (implicit in 13-03 wiring) connects them.

**Output:** 11 files created/modified; all pass their own acceptance checks locally via `npm run <script>`.

2. Write `scripts/detect-stale-refs.cjs` — scanner that:
   - Reads `reference/DEPRECATIONS.md` if present; if absent, creates it with the following minimal authoritative list before scanning:
     ```markdown
     # Deprecated Namespaces and Names

     The following references are deprecated and MUST NOT appear in shipped files.
     CI fails the build if any occurrence is detected by `scripts/detect-stale-refs.cjs`.

     ## Stale command namespaces
     - `/design:*` — replaced by `/gdd:*` in Phase 1

     ## Stale agent names
     - `design-context-builder` — split into `design-context-builder` + separate agents in Phase 3
     - `design-pattern-mapper` as single blob — replaced by concern-classifier split in Phase 3

     ## Stale stage names
     - `scan` stage — folded into `/gdd:explore` in Phase 2
     - `discover` stage — folded into `/gdd:explore` in Phase 2
     ```
   - Scans `**/*.md` (excluding `.planning/`, `.claude/`, `node_modules/`, `.design/`, `test-fixture/`, and `reference/DEPRECATIONS.md` itself) for each deprecated token.
   - Deprecated tokens (match via regex — literal string + word boundary):
     - `/design:[a-z-]+` (the `/design:` namespace)
     - `design-context-builder` (as a standalone word — flag only when not preceded by `design-context-builder-v2` style — simple approach: flag every occurrence but exclude lines containing `Phase 3` or `(replaced)` documentation markers; or just report as warnings. Simplest: flag every occurrence; then require DEPRECATIONS.md itself be excluded via the skip-list above.)
     - `design-pattern-mapper` when followed by whitespace (legacy single-blob ref; fine when followed by `-concern-classifier` — so regex is `design-pattern-mapper(?![a-z-])`)
     - `\bscan/SKILL\.md\b` and `\bdiscover/SKILL\.md\b` (stage names)
   - Output format: `{file}:{line}: {matched-token}`
   - Exits 0 if zero matches, 1 otherwise. Print `summary: <N> files scanned, <M> stale refs found`.
