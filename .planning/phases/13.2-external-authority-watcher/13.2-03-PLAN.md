```yaml
---
name: gdd-watch-authorities
description: "Fetches the design-authority feed whitelist, diffs against .design/authority-snapshot.json, and writes .design/authority-report.md (consumed by /gdd:reflect). Authority monitoring only — no trend-watching."

```

Rationale for tool set:
- `Read` — reads `.design/authority-snapshot.json` (for diagnostic footer) and STATE.md.
- `Write` — skill does not write directly; included for future proposal→confirm extensions. May be omitted if unused.
- `Task` — spawns `design-authority-watcher` agent.
- `Bash` — probes scheduled-tasks MCP via `ToolSearch` equivalent, runs grep, derives dates. (Claude Code skills use `ToolSearch` via Task; but Bash is also allowed for date math on `--since` validation.)

(If omitting `Write` passes size-budget + lint more cleanly, omit it — skill behaves correctly with `Read, Task, Bash`.)

<!-- Flag surface (locked via CONTEXT.md D-23) -->

| Flag | Effect | Agent side |
|---|---|---|
| `--refresh` | Force re-seed snapshot from current feed state without surfacing anything | Passed through to the watcher |
| `--since <ISO date>` | Surface entries newer than <date> regardless of snapshot state | Passed through to the watcher |
| `--feed <feed-id>` | Limit fetch loop to a single feed (debugging) | Passed through to the watcher |
| `--schedule <cadence>` | Register a cron via scheduled-tasks MCP; on MCP absence print a one-line diagnostic and exit 0 | Handled in the SKILL, never passed to watcher |

Valid cadence values for `--schedule`: `weekly`, `daily`, `monthly`. Anything else → print `Unknown cadence: <value>. Use one of: weekly, daily, monthly.` and exit.

<!-- Scheduled-tasks MCP probe (locked via CONTEXT.md D-24) -->

The skill probes for the MCP via Claude Code's ToolSearch equivalent:
```
ToolSearch({ query: "scheduled-tasks", max_results: 3 })

→ Non-empty → proceed to register the cron via the MCP's documented registration tool.
```

(Exact registration tool name depends on the MCP; implementer queries it at runtime via ToolSearch and follows the MCP's schema. If the MCP is absent the skill never reaches that branch.)
