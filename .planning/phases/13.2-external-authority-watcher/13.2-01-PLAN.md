<!-- Locked whitelist entry format (from CONTEXT.md D-09) -->

Each feed entry in reference/authority-feeds.md MUST follow this markdown shape:

```markdown
- **[<Human-readable title>](<homepage-url>)** — `kind: <kind-id>` · `url: <feed-url>` · `cadence-hint: <daily|weekly|monthly|quarterly|irregular>` · *<one-line rationale>*
```

Valid `kind` values (no others allowed without agent update):
- `spec-source`        — normative specs / guidelines (WAI-ARIA, Material 3 release notes, Apple HIG, Fluent 2, W3C CG)
- `component-system`   — open-source design system release notes (Radix, shadcn/ui, Polaris, Carbon, Primer, Atlassian, Ant, Mantine)
- `research`           — research institutions (NN/g articles RSS, Laws of UX GitHub releases, Baymard content updates)
- `named-practitioner` — curated blogs from individual writers with durable / spec-adjacent output
- `arena`              — user-added Are.na channel (`https://api.are.na/v2/channels/<slug>/contents`)

Locked snapshot shape (CONTEXT.md D-12) — schema MUST describe this and nothing extra:

```json
{
  "version": 1,
  "generated_at": "<ISO8601 datetime>",
  "feeds": {
    "<feed-id>": {
      "last_fetched_at": "<ISO8601 datetime>",
      "etag": "<optional string>",
      "entries": [
        { "id": "<guid-or-url string>", "hash": "<sha256 hex string>" }
      ]
    }
  }
}
```

Constraints:
- `version` MUST be the integer `1` (bump only via explicit phase).
- `feeds` is an object keyed by a stable feed-id string (kebab-case; watcher derives from the whitelist entry title).
- `entries` MAY be an empty array (first-run seeded state per D-15).
- `hash` MUST be a 64-character lowercase hex string (sha256 of `title + "\n" + summary`).
- The snapshot MUST NOT allow additional top-level properties — prevents drift from the D-12 contract.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PR author → whitelist | Any PR author can propose adding a feed. Rejected-kinds test is the structural guard against anti-slop drift. |
| External feed publisher → snapshot | Publishers control entry title/summary. Hash is over content only, so malicious publishers cannot forge diffs retroactively, but they can cause entries to appear "new" by changing titles. This is acceptable — real content drift should surface. |
