<!-- Classification decision table (locked via CONTEXT.md D-17) -->

The agent body MUST encode this table verbatim (markdown):

| Source kind | Default classification |
|---|---|
| spec-source | `spec-change` if title matches `/(added\|updated\|deprecated\|removed\|new)/i` else `pattern-guidance` |
| component-system | `pattern-guidance` if title matches new-component regex (`/(new\|add(ed)?\|introduc(e\|ing))/i` + component noun) else `craft-tip` |
| research | `heuristic-update` if title/summary mentions `/principle\|law\|heuristic\|usability finding/i` else `craft-tip` |
| named-practitioner | `craft-tip` by default; `pattern-guidance` if the entry links to a spec-source URL |
| arena | `pattern-guidance` (user-curated references) |
| any, flagged promo/newsletter/ad or matching skip-regex | `skip` |

<!-- Report file shape (locked via CONTEXT.md D-21) -->

```markdown
# Authority Report — <ISO date>

N entries surfaced across M feeds. K skipped.

## spec-change (X)
- **[Title](url)** — feed: <feed-title> — *rationale sentence*

## heuristic-update (X)
- ...

## pattern-guidance (X)
## craft-tip (X)

---

**Skipped:** K entries (see `.design/authority-snapshot.json` for the full trail).

<footer if any per-feed errors>
Fetch notes:
- <feed-id>: <error description, non-blocking>
</footer>
```

Sections are omitted when empty. Classification buckets ordered by weight: spec > heuristic > pattern > craft.

<!-- First-run behavior (locked via CONTEXT.md D-15) -->

If `.design/authority-snapshot.json` does not exist AND `--since <date>` is absent:
- Fetch every feed.
- For each feed, record the current entry set into the snapshot without comparing against anything.
- Do NOT write `.design/authority-report.md` (or write an empty one with a single header line — implementer chooses, both satisfy D-15).
- Print to stdout: `Seeded snapshot for N feeds — next run will surface new entries.`

If `--since <date>` is passed on first run, treat `<date>` as the effective "last seen" boundary and surface entries newer than that date.

<!-- Snapshot contract (locked via CONTEXT.md D-12, validated by Plan 13.2-01's schema) -->

The agent MUST write `.design/authority-snapshot.json` in the shape described in `reference/schemas/authority-snapshot.schema.json`:
- `version: 1`
- `generated_at`: ISO8601 string
- `feeds`: object keyed by feed-id; each value carries `last_fetched_at`, optional `etag`, and `entries` array capped at 200 (D-14)
- Each entry: `{ "id": <guid-or-url>, "hash": <sha256(title + "\n" + summary) hex> }`

Prune: when writing a feed's `entries`, keep only the most recent 200 by append order.
