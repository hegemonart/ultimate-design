## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 — Missing Critical Functionality] Wire authority-snapshot schema into validator PAIRS**

- **Found during:** Task 2 (Schema authoring)
- **Issue:** `scripts/validate-schemas.cjs` uses a hardcoded `PAIRS` array, not auto-discovery of `reference/schemas/*.schema.json`. CONTEXT.md §D-12 explicitly states the new schema should be "picked up automatically" by Phase 13's schema-lint. Without wiring, the schema ships but is never compiled by CI — syntax bugs introduced by downstream plans would go undetected, and the threat-model mitigation for T-13.2-03 (DoS via unbounded snapshot growth, enforced by `maxItems:200`) would have no CI gate.
- **Fix:** Added an entry for `authority-snapshot` to the `PAIRS` array in `scripts/validate-schemas.cjs`. `data: null, required: false` mirrors the existing `intel` entry's treatment (snapshot is gitignored runtime output under `.design/`). Schema is now compiled — not validated against data — on every CI run.
- **Files modified:** `scripts/validate-schemas.cjs`
- **Verification:** `node scripts/validate-schemas.cjs` now reports 6 pairs checked, 0 failures; output shows `[OK] authority-snapshot via ajv-compile`.
- **Committed in:** `f372253` (Task 2 commit)

**2. [Rule 1 — Bug] ajv-cli strict mode rejected `format: "date-time"`**

- **Found during:** Task 2 (first validator run after wiring)
- **Issue:** The plan explicitly required `format: "date-time"` on `generated_at` and `last_fetched_at` (CONTEXT D-12, plan acceptance criteria). ajv-cli v5 in default strict mode treats unknown formats as fatal errors, not warnings — the validator failed with `unknown format "date-time" ignored in schema at path "#/properties/generated_at"`. Dropping the format would violate plan acceptance. Using `--validate-formats=false` would degrade the schema to documentation-only. Both unacceptable.
- **Fix:** Updated `runAjv()` to invoke npx with `-p ajv-cli@5 -p ajv-formats@3 ajv` (installing the formats plugin alongside the CLI) and append `-c ajv-formats` to every invocation. This makes ajv *enforce* the declared formats rather than ignoring them. Benign no-op for plugin/marketplace/hooks/config/intel (none declare formats).
- **Files modified:** `scripts/validate-schemas.cjs`
- **Verification:** `node scripts/validate-schemas.cjs` exits 0 with all 6 pairs OK. Manual fuzz tests confirmed that a well-formed snapshot validates, `version: 2` is rejected (const violation), and a feed with 201 entries is rejected (maxItems violation).
- **Committed in:** `f372253` (Task 2 commit)
