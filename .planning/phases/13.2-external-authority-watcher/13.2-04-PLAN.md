```markdown
## Authority Watcher

Subscribe to a curated whitelist of design-authority sources and feed only genuinely new, classified entries into the Self-Improvement reflector. Authority monitoring — not trend watching.

### Usage

```bash
# On-demand diff + classify
/gdd:watch-authorities

# Force re-seed snapshot (recovery)
/gdd:watch-authorities --refresh

# Surface backlog since a specific date
/gdd:watch-authorities --since 2026-01-01

# Limit to a single feed (debugging)
/gdd:watch-authorities --feed wai-aria-apg

# Schedule weekly runs (requires scheduled-tasks MCP)
/gdd:watch-authorities --schedule weekly
```

### What the whitelist covers

See [`reference/authority-feeds.md`](reference/authority-feeds.md). Feeds are grouped by:

- **Spec sources** — WAI-ARIA APG, Material 3, Apple HIG, Fluent 2, W3C Design Tokens CG
- **Component systems** — Radix, shadcn/ui, Polaris, Carbon, Primer, Atlassian, Ant, Mantine
- **Research institutions** — NN/g, Laws of UX, Baymard
- **Named practitioners** — ~12 writers filtered for spec-adjacent, durable, original analysis
- **User-added Are.na channels** — extensibility point; add your own via PR to the Are.na section

### What is explicitly rejected

No Dribbble. No Behance. No LinkedIn. No generic "trending" aggregators. See `reference/authority-feeds.md` §"Rejected kinds" — enforced by `scripts/tests/test-authority-rejected-kinds.sh`.

### How the report feeds reflection

The watcher writes `.design/authority-report.md`. `/gdd:reflect` reads it alongside internal telemetry and proposes reference-file updates. Nothing auto-ships — user reviews every proposal via `/gdd:apply-reflections`.
```

  ```bash
  #!/usr/bin/env bash
  # test-authority-rejected-kinds.sh
  #
  # Asserts that reference/authority-feeds.md does NOT contain trend-aggregator
  # hosts OUTSIDE the explicit "## Rejected kinds" section. Enforces the anti-slop
  # thesis structurally (CONTEXT.md D-08, D-28).
  #
  # Exit 0 = clean. Exit 1 = at least one rejected host appeared in a live feed entry.

  set -euo pipefail

  WHITELIST="reference/authority-feeds.md"
  if [ ! -f "$WHITELIST" ]; then
    echo "FAIL: $WHITELIST not found." >&2
    exit 1
  fi

  # Split the file at the "## Rejected kinds" heading. Everything BEFORE it is the
  # active whitelist; everything AFTER is the rejection manifest (which is allowed
  # to mention these hosts — that's the whole point).
  ACTIVE_SECTION="$(awk '/^## Rejected kinds/{exit} {print}' "$WHITELIST")"

  REJECTED_PATTERNS=(
    'dribbble\.com'
    'behance\.net'
    'linkedin\.com'
    'medium\.com/topic'
    'producthunt\.com/posts'
    'top[[:space:]]*10[[:space:]]*ui'
    'trending-ui'
  )

  FAIL=0
  for pat in "${REJECTED_PATTERNS[@]}"; do
    if echo "$ACTIVE_SECTION" | grep -iEq "$pat"; then
      echo "FAIL: rejected pattern '$pat' matched in active whitelist section of $WHITELIST." >&2
      FAIL=1
    fi
  done

  if [ "$FAIL" -ne 0 ]; then
    echo "" >&2
    echo "The whitelist must not contain trend-aggregator hosts outside the '## Rejected kinds' block." >&2
    echo "See .planning/phases/13.2-external-authority-watcher/13.2-CONTEXT.md §D-08." >&2
    exit 1
  fi

  # Also assert the rejected-kinds section itself is present — regression against
  # someone deleting the section entirely.
  if ! grep -q '^## Rejected kinds$' "$WHITELIST"; then
    echo "FAIL: '## Rejected kinds' section is missing from $WHITELIST." >&2
    exit 1
  fi

  echo "OK: reference/authority-feeds.md passes rejected-kinds check."
  exit 0
  ```

  ### Test script: `scripts/tests/test-authority-watcher-diff.sh`

  ```bash
  #!/usr/bin/env bash
  # test-authority-watcher-diff.sh
  #
  # End-to-end test: runs the watcher against the frozen fixture set at
  # test-fixture/authority-feeds/, diffs the output against
  # test-fixture/baselines/phase-13.2/authority-report.expected.md.
  #
  # Uses a file:// URL override so WebFetch is not exercised in CI.

  set -euo pipefail

  FIXTURE_DIR="test-fixture/authority-feeds"
  BASELINE="test-fixture/baselines/phase-13.2/authority-report.expected.md"
  WORK="$(mktemp -d)"
  trap 'rm -rf "$WORK"' EXIT

  if [ ! -d "$FIXTURE_DIR" ]; then
    echo "FAIL: fixture dir $FIXTURE_DIR missing." >&2
    exit 1
  fi
  if [ ! -f "$BASELINE" ]; then
    echo "FAIL: baseline $BASELINE missing." >&2
    exit 1
  fi

  # Spawn the watcher in "offline fixture mode":
  #   1. Point DESIGN_DIR at a temp directory so .design/authority-snapshot.json
  #      starts absent (triggers first-run seeding — but first-run seeds silently,
  #      so we run TWICE: first run seeds, second run surfaces against the snapshot).
  #   2. Pass AUTHORITY_FEEDS_OVERRIDE=$FIXTURE_DIR so the watcher reads fixture
  #      files instead of fetching live feeds. (The watcher agent documents this
  #      environment-variable override contract in Plan 13.2-02.)
  #
  # If the watcher is not runnable from a pure shell (it's a Claude Code agent),
  # this test falls back to invoking a wrapper: scripts/tests/run-watcher-agent.cjs
  # (authored in this same task below) which uses the Claude Code SDK to spawn the
  # agent against a synthesized prompt.
  #
  # For now: the simplest testable surface is the REPORT FILE — the fixture pipeline
  # produces a known report; we byte-compare it against the baseline.

  # Run the watcher. Implementation detail — either an SDK wrapper or a direct
  # invocation depending on how the agent runtime is exposed in CI.
  # If the agent cannot be exercised in CI (MVP constraint), this test becomes
  # a BASELINE-PRESENT structural check only: it confirms the baseline exists and
  # is non-empty, and confirms the five classification section headers appear in
  # expected order. Full byte-diff is gated on the agent-runtime-in-CI follow-up.

  node scripts/tests/run-watcher-agent.cjs \
    --fixtures "$FIXTURE_DIR" \
    --work-dir "$WORK" \
    --snapshot "$WORK/authority-snapshot.json" \
    --report "$WORK/authority-report.md"

  if [ ! -f "$WORK/authority-report.md" ]; then
    echo "FAIL: watcher did not produce $WORK/authority-report.md." >&2
    exit 1
  fi

  # Normalize the header date line before diffing (the only intentionally-variable
  # line in the whole report).
  sed -E 's/^# Authority Report — .*/# Authority Report — <DATE>/' "$WORK/authority-report.md" > "$WORK/normalized.md"

  if ! diff -u "$BASELINE" "$WORK/normalized.md"; then
    echo "" >&2
    echo "FAIL: watcher output differs from frozen baseline." >&2
    echo "If the change is intentional, regenerate with: cp $WORK/normalized.md $BASELINE" >&2
    exit 1
  fi

  echo "OK: watcher-diff test passed."
  exit 0
  ```
