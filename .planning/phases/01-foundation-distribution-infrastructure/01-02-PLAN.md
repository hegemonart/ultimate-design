**Transformation rules (apply to EVERY migrated line):**
1. Add `-E` to grep flags (e.g., `grep -rn` → `grep -rEn`; `grep -roh` → `grep -rEoh`; `grep -rln` → `grep -rEln`)
2. Replace every `\|` with `|`
3. Where literal CSS parens must match (e.g., `oklch(...)`, `hsl(...)`, `rgb(...)`), escape them: `oklch\([^)]*\)` (because in `-E` mode bare `(` becomes a grouping operator — see Pitfall 1)
4. Where `\(group\)` was used purely for grouping (e.g., `font-\(thin\|light\|normal\)`), either drop the escaping (use `font-(thin|light|normal)`) or rewrite without grouping — be consistent per pattern
5. Keep `[^)]` character classes unchanged — they are unaffected by `-E`
6. Keep all other flags (`-r`, `-n`, `-o`, `-h`, `-l`, `--include=`, `2>/dev/null`, `| sort | uniq -c | sort -rn | head`) unchanged

```bash
# AFTER (POSIX):

```
